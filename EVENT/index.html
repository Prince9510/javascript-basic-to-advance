JavaScript Event Listeners Cheatsheet

üëÇ Common Event Listeners

Events are actions that happen in the system you are programming. The system fires a signal when an event occurs, allowing you to run code in response. Here are some of the most common events.

üñ±Ô∏è Mouse Events

Event Type Description Key Event Properties Example
click Fires when the user clicks on an element. clientX, clientY, button, target element.addEventListener('click', (e) => {});
dblclick Fires when the user double-clicks on an element. clientX, clientY, button, target element.addEventListener('dblclick', (e) => {});
mousedown Fires when a user presses a mouse button over an element. button element.addEventListener('mousedown', (e) => {});
mouseup Fires when a user releases a mouse button over an element. button element.addEventListener('mouseup', (e) => {});
mousemove Fires when the mouse pointer is moving over an element. clientX, clientY element.addEventListener('mousemove', (e) => {});
mouseover Fires when the pointer moves onto an element or its children. relatedTarget element.addEventListener('mouseover', (e) => {});
mouseout Fires when the pointer moves out of an element or its children. relatedTarget element.addEventListener('mouseout', (e) => {});
mouseenter Fires when the pointer moves onto an element. (Does not bubble). relatedTarget element.addEventListener('mouseenter', (e) => {});
mouseleave Fires when the pointer moves out of an element. (Does not bubble). relatedTarget element.addEventListener('mouseleave', (e) => {});
contextmenu Fires on right-click to open a context menu. preventDefault() element.addEventListener('contextmenu', (e) => {});

üìù Form & Input Events

Event Type Description Key Event Properties Example
submit Fires when a <form> is submitted. target, preventDefault() form.addEventListener('submit', (e) => {});
input Fires when the value of an <input>, <select>, or <textarea> changes. target.value, data inputEl.addEventListener('input', (e) => {});
change Fires when the value is committed (e.g., after losing focus). target.value selectEl.addEventListener('change', (e) => {});
focus Fires when an element receives focus. target inputEl.addEventListener('focus', (e) => {});
blur Fires when an element loses focus. target inputEl.addEventListener('blur', (e) => {});
invalid Fires when an input fails validation. target inputEl.addEventListener('invalid', (e) => {});
